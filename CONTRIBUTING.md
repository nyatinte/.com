# コントリビューションガイド

このプロジェクトへの貢献ありがとうございます！

## コミットメッセージのルール

このプロジェクトでは、**コミットメッセージは日本語で記述**します。

### コミットメッセージの形式

```
簡潔な変更の要約（50文字以内を推奨）

- 変更内容の詳細1
- 変更内容の詳細2
- 変更内容の詳細3
```

### 良いコミットメッセージの例

```
ユーザー認証機能を実装

- ログインフォームコンポーネントを追加
- JWT認証のバックエンドAPIを実装
- 認証状態管理用のContextを作成
```

```
パフォーマンスを改善

- 画像の遅延読み込みを実装
- 不要な再レンダリングをメモ化で防止
```

### コミットの粒度

- **1つのコミットには1つの論理的な変更**を含める
- 複数の機能追加がある場合は、機能ごとにコミットを分割する
- リファクタリングと機能追加は別々のコミットにする

## プルリクエストのルール

### タイトル

プルリクエストのタイトルも**日本語で記述**します。

例：
- `ユーザープロフィール編集機能を追加`
- `モバイル表示のレイアウトを修正`
- `依存関係を更新`

### 説明

プルリクエストの説明には以下を含めてください：

```markdown
## 概要
この変更の目的と背景を記述

## 変更内容
- 変更点1
- 変更点2
- 変更点3

## テスト
- テストした内容
- 確認した動作

## スクリーンショット（該当する場合）
UI変更の場合はスクリーンショットを添付
```

### プルリクエストの粒度

- **1つのPRには1つの機能または修正**を含める
- 大きな機能は複数のPRに分割する
- レビューしやすいサイズ（変更行数300行以内を目安）に保つ

## 開発フロー

1. mainブランチから新しいブランチを作成
   ```bash
   git checkout -b feature/your-feature-name
   ```

2. 変更を加えてコミット
   ```bash
   git add .
   git commit -m "変更内容を日本語で記述"
   ```

3. リモートにプッシュ
   ```bash
   git push -u origin feature/your-feature-name
   ```

4. プルリクエストを作成

## パッケージマネージャー

このプロジェクトでは **bun** を使用しています。

```bash
# 依存関係のインストール
bun install

# 開発サーバーの起動
bun run dev

# ビルド
bun run build

# リント
bun run lint
```

## コーディング規約

- TypeScriptの型定義を適切に使用する
- ESLintのルールに従う
- コンポーネントは機能ごとに分割する
- 適切なコメントを日本語で記述する
